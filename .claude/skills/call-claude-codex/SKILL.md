# call-claude-codex — 在指定目录启动子 Claude Code / codex 干编程活

把一个**独立的 AI 编程 agent**（claude 或 codex）以 **headless 非交互模式**放到**用户指定的工作目录**里跑，跑完收回结果。用于：并行改多文件、worktree 隔离、或想要 codex（不同厂商）的独立视角。

> **每次真正调用子 CLI 前，都必须先向用户发起一次"二次确认"——硬性步骤，不跳过。** 两条警告（可用性、`-p` 收费风险）就**固定写在这条确认消息里**，不单独堆在开头——这样主流程干净，警告又每次实战必现。见第 3 步。

## 第 1 步：确认 claude / codex 可用（只读探测）

先查要用的那个 CLI 是否已安装、能跑：

```powershell
# 二选一，看用户要调哪个
claude --version
codex --version
```

- 命令不存在 → **别往下走**，直接告诉用户"X 未安装 / 不在 PATH"，让用户先装。
- 能出版本号 ≠ 已登录能干活。是否已登录 / 有额度，实战调用时由返回结果体现；首次若报认证错误，如实转告用户去登录，不硬试。

## 第 2 步：选定工具 + 目标工作目录（cwd）

- **工具**：claude 还是 codex？听用户的；没指定就默认 claude。
- **目标工作目录**：claude 的主工作区 = 启动它时的 shell 当前目录；codex 用 `--cd`。让用户明确要进哪个目录（如 `D:\某项目`）。
- **任务描述**：要具体、自包含——子进程是独立会话，看不到我们这段对话的上下文。

## 第 3 步：发起二次确认（硬性，带警告）——核心步骤

**真正启动子 CLI 之前**，向用户发一条确认消息，等用户点头才跑。确认消息**必须包含**以下内容（照此模板，缺一不可）：

```
我将调用【claude / codex】，在工作目录【<绝对路径>】执行：
<一句话任务描述>

⚠️ 前提：claude / codex 需处于可用状态（已安装、能跑、已登录/有额度）；不可用会如实回报，不硬试。
⚠️ 成本：Claude Code 的 -p（headless）模式官方原计划收费，目前已暂停该收费，但"暂停 ≠ 取消"，未来可能恢复；codex 走各自账号额度。
[仅当跨出本仓库时加这一行] ⚠️ 越界：目标目录在 本仓库根目录 之外，本次为经你单次授权的例外，仅限该目录、仅限当次。

确认执行吗？（回复"确认"或给修改）
```

要点：

- **两条警告（前提、成本）每次都固定带上**，无论是否跨仓库——这是让用户每次都看到风险的设计，不是可选项。
- 跨仓库时**额外**带越界提示（呼应 CLAUDE.md 的"绝对禁止 + 二次授权例外"）。
- 用户没明确确认前，**绝不**启动。

## 第 4 步：用 headless 模式启动（绝不交互式 TUI）

**只走非交互模式**。裸 `claude` / `codex` 进交互界面会在子进程里等 stdin、挂起，**绝对不要**那样调。

```powershell
# claude：用 Push-Location 设主工作区，跑完自动还原
Push-Location "<目标绝对路径>"
try {
  claude -p "<任务描述>"
} finally {
  Pop-Location
}

# 或不切当前目录，直接给子进程指定工作目录
Start-Process claude -ArgumentList '-p "<任务>"' -WorkingDirectory "<目标绝对路径>" -NoNewWindow -Wait

# codex：自带 --cd
codex exec --cd "<目标绝对路径>" "<任务描述>"
```

- 子 CLI 是**独立会话**，不继承本仓库 CLAUDE.md（目标目录在仓库外时它没有这条边界约束），所以边界要在第 3 步由我们把好关。
- 长任务可后台跑（`run_in_background` 或 `Start-Process` 不阻塞），完成后再收结果。

## 第 5 步：收回结果、还原、汇报

- 拿子 CLI 的 stdout 结果，**完整转述给用户**（用户只看得到飞书那条回复，结果要在正文里写全，不能依赖"终端里说过"）。
- 确保 `Push-Location`/`Pop-Location`（或 Start-Process）把我自己的工作目录还原回仓库根。
- 把这轮回复写入 `workspace/.outbox/reply.md`（桥接的收尾信号）。

## 边界

- 启动子 AI 进程 = 外部操作 + 消耗额度，**一律先二次确认**（第 3 步），不擅自跑。
- 跨出 `本仓库根目录` 的工作目录，严格走 CLAUDE.md 的"绝对禁止 + 二次授权例外"，单次单范围，不扩大。
- 自己能干的简单编程活儿**别套娃**（成本翻倍、独立会话无上下文）；本 skill 只在确需"独立 agent + 指定目录"时用。
- 绝不交互式启动子 CLI（会挂起），只用 `-p` / `exec` 这类 headless 入口。
