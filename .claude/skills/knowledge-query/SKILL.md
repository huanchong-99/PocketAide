# knowledge-query — 知识查询（语义 + 关系 + 概览）

通过 kg 知识图谱 CLI 检索用户的知识库，基于**真实检索结果**回答。检索不到就老实说没有，**绝不编造**。

## 先判断查询类型，选对命令

### 精确 / 模糊 / 关联内容查询 → search
"上次那个 X 怎么用来着""我看过一个关于 Y 的东西是啥""我存的关于 A 和 B 有没有交叉"：
```bash
python tools/kg/kg.py search "<把用户问题转成检索语>" --k 5
```
返回 JSON 列表 `[{id,title,score,snippet,tags,source}]`（按 score 降序，已归一化，余弦+FTS 混合）。
- 基于命中的 `title/snippet/source` 回答；需要全文就读对应文件（`knowledge/<id>`）。
- **判定"是否真检索到"要 score + snippet 双重看，别只看分数**：bge-m3 向量各向异性会让**完全无关**的查询也拿到 0.45~0.55 的分（真正命中通常 ≥ 0.7）。所以——结果为空、或 score 普遍偏低（如都 < 0.45）、**或分数虽过线但 snippet 内容明显与问题无关** → 一律按"知识库里没有/几乎没有相关内容"如实告知边界、说清覆盖范围，**绝不硬答/编造**。

### 关系 / 关联查询 → related
"和这个主题相关的我还存过啥""X 关联到哪些东西"：
```bash
python tools/kg/kg.py related "<id或标题或关键词>" [--depth 1]
```
返回邻居（出边+入边的 wikilink 关联）。基于 `neighbors` 回答关联网络。找不到起点节点会 stderr 报 error + 非零退出码——这时改用 search。

### 概览查询 → overview
"我知识库里关于 AI 的内容有多少、大概覆盖哪些方面""我一共存了多少知识"：
```bash
python tools/kg/kg.py overview [--tag <标签>]
```
返回 `total_nodes/by_tag/by_subdir/by_type/total_links`。基于这些计数给用户一个覆盖概览。

## 回答原则
- **基于检索结果回答**，不靠你自己的记忆冒充用户的知识库内容。
- 引用时点明来源（哪篇笔记 / source），让用户能溯源；必要时附 `[[笔记标题]]`。
- 检索不足如实说，并可建议用户换个说法再查、或先把相关内容摄入（转 knowledge-write）。
- 多个匹配时列出让用户挑，别擅自只取一个就当定论。

## 边界
- 命令输出 UTF-8 JSON，直接解析；非零退出码读 stderr 的 `error`。
- 首次调用会拉起常驻嵌入进程（稍慢），之后很快。
- 只读知识库、不改写；要写/改知识走 knowledge-write。只在本仓库范围活动。
