# knowledge-write — 知识摄入（原子化 + 四段式 + 二次确认 + kg 索引）

把零散内容（对话、网页、视频转写、文档片段）沉淀成**原子知识笔记**写入 `knowledge/`，并更新 kg 索引。
两条核心原则贯穿全程：

1. **原子化**：一篇笔记只讲**一个足够窄的主题**。主题拆得细——同一大主题多次汇入时，并进对应窄笔记或新开一篇，用 `[[wikilink]]` 串，避免单文件无限膨胀。**硬上限：单篇超出"只讲一件事"就拆分，绝不让它越长越大。**
2. **二次确认**（硬规则）：整理好先发用户确认"整理好了，是这个吗？要不要改？"，**确认通过才写盘**。要改就改完再确认，循环到满意。**批量摄入除外**——先确认范围、后抽查。

## 流程

### 1. 判断主题并原子化拆分
一段输入若含多个独立知识点，**拆成多篇**，各自成原子笔记，互相 `[[wikilink]]`。
判据：这篇能否用一句话概括"只讲这一件事"？不能就再拆。

### 2. 查重（防膨胀关键，落盘前必做）
对每个待写主题跑语义检索，找已有的同主题笔记：
```bash
python tools/kg/kg.py search "<该主题的关键词>" --k 5
```
解析返回的 JSON（`[{id,title,score,snippet,tags,source}]`）：
- **有命中且属同一窄主题**（score 高、主题吻合）→ 读那篇、**增量并入**：只补真正新增的要点/原文，去重；并入后若超出"只讲一件事"就**拆分**成多篇。
- **是同大主题的不同侧面** → **新开一篇**，`[[wikilink]]` 关联，别塞旧文件。
- **无命中** → 新建。

### 3. 组织成四段式
每篇笔记固定四块（按易读顺序），frontmatter 用系统时钟取日期（`Get-Date`，别猜）：

```markdown
---
type: knowledge
tags: [主题, 关键词]
source: 来源(URL/对话/视频链接/文档名)
created: YYYY-MM-DD
---

# 标题（这一个原子主题）

## 摘要
提炼后的核心要点（bullet，客观）。

## 参谋解读
你的视角：为什么重要、对用户/项目的启发、注意点、关联。

## 原文片段
关键原文引用 / 代码原句（可溯源，只留要紧的，别全文照搬）。

## 参考
出处链接 / 文档名 / 对话时间等。

[[关联笔记1]] [[关联笔记2]]
```

### 4. 二次确认
- **单条**：把整理好的笔记（含将写入的路径、新建还是并入哪篇）发用户确认；要改就改完再确认；通过才写。
- **批量**：先说明"我打算摄入这 N 条、分到这些主题/目录"，确认范围；写完抽几条给用户验。

### 5. 写入
路径 `knowledge/<子目录>/YYYY-MM-DD-主题关键词.md`，按主题归子目录（`ai/` `security/` …，没合适的就建有意义的新子目录）。命名一眼可辨主题。

### 6. 版本控制
写入后 `PostToolUse` 钩子自动 `git add + commit`，无需手动 commit。push 由桥接层统一做。

### 7. 更新 kg 索引（必做）
写完每篇后增量索引：
```bash
python tools/kg/kg.py index --file <相对 knowledge 的路径>
```
（一次写多篇可最后跑一次 `python tools/kg/kg.py index --all`。）解析 stdout JSON 看 `indexed` 确认生效；非零退出码读 stderr 的 `error` 排查。

## 边界
- 只往 `knowledge/` 写知识主体；临时产物归 `workspace/`。只在本仓库范围活动。
- 拿不准主题归类/是否重复时，宁可问用户，别乱建乱并。
- kg 命令输出 UTF-8 JSON，直接解析；首次调用会拉起常驻嵌入进程（稍慢），之后很快。
